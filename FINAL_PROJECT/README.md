# REST API для мобильного приложения ФСТР

REST API для отправки, хранения и получения данных о горных перевалах, включая информацию о пользователе, координатах, уровне сложности и изображениях.

## Особенности

- Регистрация и валидация пользователей с уникальными email и телефоном.
- Валидация координат (широта, долгота, высота).
- Уровни сложности перевала по сезонам (зима, лето, осень, весна).
- Загрузка изображений перевала.
- Статусы обработки заявок: новая, на проверке, принята, отклонена.
- CRUD‑операции для данных о перевалах.
- Поиск перевалов по email пользователя.

## Технологии

- **Backend:** Python, Django, Django REST Framework.
- **Документация API:** drf‑spectacular (OpenAPI/Swagger).
- **База данных:** реляционная (PostgreSQL).
- **Валидация:** встроенные валидаторы Django (`RegexValidator`, `MinValueValidator`, `MaxValueValidator`).
- **Используемые зависимости:** [requirements.txt](https://github.com/Darya-20/PythonLearningProjects/blob/master/FINAL_PROJECT/requirements.txt)

## Использование API
**Доступные эндпоинты**
| Метод | Эндпоинт | Описание |
|-------|----------|---------|
| `POST` | `/submit_data/` | Отправка данных о новом перевале |
| `GET` | `/pass/<int:pk>/` | Получение полной информации о перевале по ID |
| `PATCH` | `/update_pass/<int:pk>/` | Редактирование данных перевала (только если статус `new`) |
| `GET` | `/list_by_user_email/` | Получение списка перевалов пользователя по email |
